# Iterations

The main thread records each completed loop here.

An iteration is not just a progress report. It is the handoff from the previous
system state into the next design cycle.

Every iteration should answer:

- What can the system do now?
- What changed in this round?
- What did review find?
- What assumptions were validated or rejected?
- What should the main thread design next?
- Which todos were added, completed, blocked, or deferred?

Use `template.md` for new iteration notes.
